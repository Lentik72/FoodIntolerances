## 1. Decisions at a glance

| Decision | Choice |
|---|---|
| Navigation | 4 tabs + center capture button: **Home · Timeline · [+] · Insights · Health** |
| Capture | **One smart sheet** from [+]: voice on top, learned one-tap chips, category buttons |
| Home | **Today-first dashboard**; assistant is one card, not the whole screen |
| Insight strength | **Word-scale badge + observation dots** (percent lives in drill-down only) |
| Onboarding | **Value-first**: two asks before Home; HealthKit backfill staged as the wow moment |
| Visual direction | **Calm clinical**: paper background, serif headings, deep teal accent |

## 2. Navigation & screen map

```
TabBar:  Home | Timeline | [+] | Insights | Health

Home (Today)
├─ "Feeling off?" bar → What Worked flow (symptom picker → ranked past interventions)
├─ Due now: dose rows (tap = logged + undo toast)
├─ Assistant card (one card: check-ins, contextual nudges, tappable replies)
├─ Newest insight teaser → Insights detail
├─ Passive health strip (sleep · steps · active mission progress)
└─ First-week only: backfill summary card ("your sleep averages 6h 41m")

Timeline
├─ Unified reverse-chron feed, day grouping, category color coding
├─ Filter chips (category, severity, source), search (FTS)
├─ Inline per-day severity sparklines
└─ Tap event → detail/edit (soft delete, correction)

[+] Capture sheet (modal, from every tab; long-press = straight to voice)
├─ 🎤 "Describe your day…" → live-parse screen (events appear as you speak,
│   per-event confidence, edit/confirm before save)
├─ Frequent chips (learned from history; tap = logged + undo toast — 2 taps total)
├─ Category buttons: Symptom (severity slider + body map) · Meal (text/photo) ·
│   Dose (from Cabinet) · Photo · Note
└─ Same sheet reachable via lock-screen widget, Action button, App Intents

Insights
├─ Active patterns (cards, §4)
├─ "No effect" findings (own section — headline feature, §4)
├─ Needs more data (blocked candidates + one-tap fixes → missions)
├─ Archive (decayed / dismissed)
└─ Card → Evidence drill-down → any event → Timeline

Health
├─ Cabinet: meds / supplements / peptides — stock, refill alerts, peptide vial
│   math (doses remaining), reorder warnings
├─ Protocols & Experiments: active (adherence rings), history, clinic-imported;
│   experiment wizard lives here + from "Test it" on insight cards
├─ Labs: trend tables/charts per analyte, import (photo/PDF/CSV)
├─ Health Confidence: data-quality % + per-category bars, missions
├─ Doctor Report: date-range picker → PDF preview → share
└─ Profile & Settings: allergies, AI/privacy controls, notifications, data export

Overlays (any screen)
├─ Red-flag interstitial ("seek care now") — takes over on red-flag symptom log
└─ Clinic QR import → protocol preview → consent → pre-configured experiment
```

## 3. Capture sheet (the friction-critical surface)

- Repeat log = **2 taps** ([+] → chip). Undo toast instead of confirm dialogs.
- Voice: mic button opens a live-parse screen — parsed events render as editable rows
  *while speaking* (on-device Foundation Models; §6.2 of product spec). Save confirms
  all; per-event parse confidence shown subtly (low-confidence rows highlighted for review).
- Chips: ranked by frequency × recency × time-of-day (morning shows breakfast + AM doses).
- New-item flows escalate gently: chip-first, full form only when something is new.

## 4. Insight cards & evidence drill-down

Card anatomy (word-scale + dots — replaces "confidence %" on cards):

```
[MODERATE PATTERN]                     ← badge: Early signal / Moderate / Strong
🥛 Dairy → bloating                    ← plain-language claim
In your last 8 dairy days,
bloating followed in 6:  ● ● ○ ● ● ● ○ ●   ← one dot per exposure; filled = outcome
usually within ~12h · avg severity +2.1      followed; hollow = it didn't; each dot
[All evidence →] [Test it] [Dismiss]         taps to that day
```

- Badge thresholds (from engine confidence): Early signal 0.3–0.5 · Moderate 0.5–0.75 ·
  Strong >0.75 (reachable only with experiment evidence, per product spec §7.5).
- Drill-down shows: every exposure→outcome pair *including misses*, confounder warnings
  ("coffee was present in 5 of 8 — can't tell these apart yet · try one without the
  other"), and raw engine numbers (ratio, confidence, lag) at the bottom for power
  users and the clinic.
- "No effect" cards use the same anatomy with an even tone — they are presented as
  wins ("you can stop wondering about this"), not failures.

## 5. Onboarding (value-first)

1. **Promise** — one screen, two sentences: "Find what actually helps you. Your data
   never leaves this device."
2. **Connect Apple Health** — framed as *import*, not permission.
3. **The wow moment** — animated live backfill: "✓ 14 months of sleep ✓ 212 workouts
   ✓ cycle data … You're not starting from zero." (progress UI mandatory; budget in
   product spec §17).
4. **"What brings you here?"** — tap main symptoms (seeds chips + red-flag list).
5. **Meds/supplements/peptides quick add** — explicitly skippable.
6. **Home** — backfill summary card + one clear mission: log today.

Clinic branch: arriving via clinic QR pre-fills the protocol and its items; step 5 is
skipped. Everything not asked here arrives later as contextual prompts (Phase 3
missing-data engine).

## 6. Visual direction — "Calm clinical"

A beautifully kept medical notebook: credibility from typography and restraint, warmth
from paper tones. Personality lives in language and type, not decoration.

**Light (default):**

| Token | Value |
|---|---|
| Background | `#FAF7F2` (warm paper) |
| Card | `#FFFFFF`, border `#E5DFD4`, faint shadow |
| Ink (text) | `#1C1B18` |
| Accent (actions, links) | `#2E7D74` deep teal |
| Evidence dots / warm alerts | `#C77E32` amber (hits), `#D8D2C6` (misses) |
| "Feeling off?" bar | `#FDF3E3` with amber border |

**Dark:** same hierarchy on `#15140F`-family warm dark tones; teal and amber re-tuned
for contrast (AA minimum). Both modes ship at launch.

**Type:** serif display (New York) for screen titles and insight claims; SF Pro for
body/UI; SF Mono only for lab values. **Dynamic Type everywhere** — layouts must
survive XXL sizes (older audience is core; product spec §17).

**Category colors (timeline coding):** one muted hue per event category, consistent
app-wide, chosen against both backgrounds (final palette in Phase 1 design pass with
the dataviz/frontend-design review).

**Doctor Report PDF stays boring-clinical** regardless of app skin (product spec §10).

## 7. Interaction principles

- Undo, never confirm: destructive-feeling actions get an undo toast, not a dialog.
- One-handed reach: primary actions in the bottom half; [+] center-thumb.
- No streak punishment anywhere (product spec §8); missions show progress, not guilt.
- Every number can be tapped to see where it came from (evidence, event, or import).
- Red-flag interstitial takes precedence over all UI states.

## 8. Open UI items (for the Phase 1 design pass)

- Final app name + icon (rebrand — product spec §17 open decisions).
- Category color palette (against both modes, color-blind safe).
- Body-map port styling to calm-clinical.
- Empty states for every screen (esp. Insights before first pattern: show "what the
  engine is watching" so pre-insight weeks feel alive).
- Widget & watch-face-adjacent layouts (App Intents surfaces).
